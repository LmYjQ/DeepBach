## 一、数据准备

### 1.1 流程概览

```
JSON简谱文件 → preprocess_chinese_score_batch.py → PT张量数据 → 训练/生成
```

### 1.2 数据格式

**输入**：JSON 格式的简谱文件，包含 `notes` 数组，每个音符有：
- `value`: 简谱数字（"1"~"7" 或 "0"休止符）
- `octave`: 八度偏移（0=中央八度，1=高八度，-1=低八度）
- `duration`: 时值（相对于 beat）
- `ban`: 是否为板音（1=固定点）

### 1.3 处理过程

1. **读取 JSON**，计算每个音符对应的 tick 位置
   - `tick_position += duration * subdivision`
   - subdivision=8 表示每拍 8 个 tick

2. **构建 chorale_tensor**：(num_samples, 1, total_ticks)
   - 每个 tick 存储 MIDI pitch
   - 简谱数字 → MIDI: `base_pitch = int(value) + 60`（中央C=60），再加 octave偏移
   - 例如：简谱 "3" octave=0 → MIDI63

3. **构建 metadata_tensor**：(num_samples, 1, ticks, 6)
   - 列顺序：`[IsPlaying, Tick, Mode, Key, Fermata, voice_id]`
   - IsPlaying: 1 表示该 tick 有音符
   - Tick: tick % subdivision（拍内位置）
   - Mode: 1 (major)
   - Key: 10 (D major)
   - Fermata: 0
   - voice_id: 0

4. **构建 note2index / index2note** 映射表
   - 将所有出现的 MIDI pitch 映射为连续索引
   - 添加特殊符号：START_SYMBOL, END_SYMBOL, REST_SYMBOL, SLUR_SYMBOL

5. **保存为 PT 文件**
   - 包含 chorale_tensor, metadata_tensor, note2index, index2note, sequence_size, subdivision

### 1.4 输出

在 `preprocessed_data/` 目录下生成 `.pt` 文件，如：
- `dataset_final8_combined_tensor_dataset.pt`
- `dataset_jin_combined_tensor_dataset.pt`

## 二、模型训练

### 2.1 流程概览

```
PT数据 → SimpleNotationDataset → DeepBach (VoiceModel×N) → 训练 → 模型保存
```

### 2.2 数据加载

**SimpleNotationDataset** 封装 PT 文件：
- `chorale_tensor`: (N, 1, ticks)音符序列
- `metadata_tensor`: (N, 1, ticks, 6) 元数据
- `note2index_dicts`: 音符→索引映射
- `sequence_size`: 上下文窗口大小（默认8 beats）
- `subdivision`: 每拍 tick 数（默认8）

**tensor_dataset** 属性返回：
- chorale: (N, ticks)
- metadata: (N, ticks, 5)  # 只取 [Tick, Mode, Key, Fermata, voice_id]

### 2.3 模型架构

**DeepBach** 包含 N 个 **VoiceModel**（N=声部数）：

```
输入层：
  - note_embedding: (num_notes, note_embedding_dim)
  - meta_embedding: (num_metas, meta_embedding_dim)

双向LSTM：
  - lstm_left: 处理左侧上下文
  - lstm_right: 处理右侧上下文
  - 隐藏大小: lstm_hidden_size
  - 层数: num_layers

输出层：
  - mlp_center: 处理中心音符
  - mlp_predictions: 拼接后映射到 num_notes
```

**上下文窗口**：
- `timesteps_ticks = sequence_size * subdivision // 2`
- 例如 sequence_size=8, subdivision=8 → timesteps_ticks=32
- 模型使用前后各 32 个 tick 作为上下文

### 2.4 模型文件命名

模型文件名格式（参数编码在文件名中）：
```
voicemodel_{suffix}_ne{val}_me{val}_lh{val}_ll{val}_ld{val}_li{val}_{voice_index}.pt
```

参数说明：
| 符号 | 含义 | 示例值 |
|------|------|--------|
| ne | note_embedding_dim | 50 |
| me | meta_embedding_dim | 25 |
| lh | lstm_hidden_size | 128 |
| ll | num_layers | 2 |
| ld | dropout_lstm | 0.5 |
| li | linear_hidden_size | 128 |

示例：`voicemodel_final8_ne50_me25_lh128_ll2_ld0.5_li128_0.pt`

### 2.5 训练过程

1. **初始化**：创建 VoiceModel，随机初始化 embedding 和 LSTM 权重
2. **迭代训练**：
   - 对每个 batch，随机选取 tick 位置
   - 提取该位置的上下文窗口
   - 前向传播计算 loss，反向传播更新权重
3. **保存模型**：保存到 `models/voicemodel_{...}_{voice_index}.pt`

## 三、乐谱生成

### 3.1 流程概览

```
JSON(ban标记) → 拆分段落 → 逐段生成(级联) → 合并保存MIDI
```

### 3.2 生成模式

#### 3.2.1 完整生成（无约束）

使用 `generate_simple_notation.py`：
- 从头生成指定长度的序列
- 使用随机初始化的 tensor
- 调用 `deepbach.generation()` 进行 Gibbs 采样

#### 3.2.2 约束生成（多段）

使用 `generate_multi_segment.py`：
- 根据 JSON 中的 `ban=1` 标记拆分段落
- 每段以相邻段的生成结果作为上下文
- 级联生成保证音乐连贯性

### 3.3 Gibbs 采样过程

**初始化**：
- 创建空白 tensor 或使用已有内容
- 对生成范围随机初始化

**迭代采样**（默认 300~500 次）：
```
for iteration in range(num_iterations):
    1. 温度退火：temperature *= 0.9993（从1.1降到1.0）
    2. 随机选取 batch_size 个 tick 位置
    3. 对每个位置：
       - 提取前后32个tick作为上下文
       - 用LSTM编码左右上下文
       - 拼接左右hidden state预测该位置音符分布
       - 从分布中采样一个音符
    4. 更新 tensor 中对应位置
```

**核心原理**：
- Pseudo-Gibbs 采样：每次更新多个位置（非串行 Gibbs）
- 模型输入：周围所有 tick 的音符+元数据
- 模型输出：该位置的条件概率分布
- 温度控制探索程度：高温=多样化，低温=确定性

### 3.4 多段级联生成

**段落拆分**：
- 根据 `ban=1` 的 tick 位置划分段落
- 每段结构：[左固定音, 待生成区域, 右固定音]

**级联流程**：
```
段0 [0, 16)：
  - 输入：位置0=左固定音，其余空白
  - 生成范围：[1, 16)
  - 输出：位置0保持，其余生成

段1 [16, 32)：
  - 输入：位置0~15=段0结果，位置16=左固定音，其余空白
  - 生成范围：[17, 32)
  - 输出：位置16保持，其余生成

...依此类推
```

**关键设计**：
- 每段的输出写入共享的 `chorale_tensor`
- 下一段读取上一段的生成结果作为上下文
- 固定音在生成后写回，确保不丢失

### 3.5 Padding 与上下文

**模型内部处理**：
```python
# padding前后各加 timesteps_ticks 个符号
tensor_chorale_padded = [START×32] + [原始] + [END×32]

# 生成范围映射
time_index_range_ticks = [gen_start + 32, gen_end + 32)
```

**上下文窗口示意**（timesteps_ticks=32）：
```
位置: -32 ~ -1 | 0 ~ 15 | 16 ~ 47 | 48 ~
内容:   START符号 | 原始 | 等待生成 | END符号
                 ↑ ↑
           生成范围外(上下文)   生成范围外(上下文)
```

## 四、命令汇总

### 4.1 数据预处理

```powershell
# 批量处理 dataset_final8 目录下的所有 JSON
python preprocess_chinese_score_batch.py --input 'D:/code/music/qmx_reader/dataset_final8'

# 处理 dataset_jin
python preprocess_chinese_score_batch.py --input 'D:/code/music/sizhu_data/data/jin'

# MIDI → JSON（反向转换）
python midi_to_json.py -i "D:\code\music\qmx_reader\dataset_midi\云庆5.9.mid" -o "D:\code\music\qmx_reader\dataset_midi2json\云庆5.9.json"
```

### 4.2 模型训练

```powershell
# 基础训练（使用默认参数）
python train_simple_notation.py --data ./preprocessed_data/dataset_final8_combined_tensor_dataset.pt --train 5 --batch_size 8

# 自定义参数训练
python train_simple_notation.py --data ./preprocessed_data/dataset_jin_combined_tensor_dataset.pt --train 100 --batch_size 16 --note_embedding_dim 50 --meta_embedding_dim 25 --lstm_hidden_size 128 --linear_hidden_size 128 --dropout_lstm 0.5 --num_layers 2
```

**训练参数说明**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--data` | 必填 | PT数据文件路径 |
| `--train` / `-t` | 10 | 训练轮数 |
| `--batch_size` / `-b` | 16 | 批次大小 |
| `--note_embedding_dim` | 100 | 音符嵌入维度 |
| `--meta_embedding_dim` | 50 | 元数据嵌入维度 |
| `--lstm_hidden_size` | 200 | LSTM隐藏层大小 |
| `--linear_hidden_size` | 200 | 线性层隐藏大小 |
| `--dropout_lstm` | 0.2 | LSTM dropout |
| `--num_layers` | 2 | LSTM层数 |
| `--models_dir` | models | 模型保存目录 |

### 4.3 乐谱生成

#### 完整生成（无约束）

```powershell
# 生成 512 ticks（约8小节）
python generate_simple_notation.py --data ./preprocessed_data/dataset_final8_combined_tensor_dataset.pt --load --output generated.mid --sequence_length_ticks 512
```

#### 多段约束生成


**生成参数说明**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--json` / `-j` | 必填 | 简谱JSON文件路径 |
| `--data` / `-d` | 必填 | PT数据文件路径 |
